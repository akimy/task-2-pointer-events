# task-2-pointer-events
### Домашнее задание.
#### Работа с сенсорным пользовательским вводом.

Преподаватель Владимир Рыбаков.  

Была реализована игра Dungeons of Doom.
Игра работает корректно только на сенсорных устройства с поддержкой multitouch events.
Первая дверь - мультинажатие на три точки - была реализована преподавателем в качестве примера.
Вторая дверь - задвижки (слайдеры) с ограничением по перемещению в одну сторону и по оси X. Их необходимо выдвинуть не достаточное расстояние чтобы они закрепились (lock).
Третья дверь - drag & drop, нужно найти секретное место с помощью света алмаза, чем зеленее алмаз - тем ближе секретное место.
Последний уровень - жесты, используются свайпы. Инструкция есть в истории.

### По просьбе Димы добавлен ход мыслей
В двери на первом этаже мы реализуем движение задвижек, (часто встречающийся кейс в бизнесе - слайдеры), ограничиваем передвижение по вертикальной оси и в одном направлении. Используем setPointerCapture для того чтобы события приходили в элемент который мы перемещаем. Начальную координату события храним в lexical enviroment функции-класса и используем для расчета разницы и transform: translateX используем для анимации. (пытался оборачивать обработку pointermove в requestAnimationFrame, но это вызвало то - что событие "застревало", см. последний коммит).

Дверь на втором этаже - drag & drop event, + рассчет по теореме пифагора расстояния между двумя точками на декартовой плоскости и изменение HSLA для смены цвета (Math.min граничное условие по максимальному расстоянию, чтобы цвет алмаза не уходил из интервала красно-зеленого спектра), на onpointerup проверяем расстояние до алмаза.

Сундук - эвент с четыремя колоннами из Might&Magic Legacy. Используется библиотека Hammer.js для детекта свайпов (события вешаются на onswipeleft, onswiperight и так далее).
